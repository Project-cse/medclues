# MediChain+ Mobile (Patient App)

React Native + Expo patient app connected to the **real FastAPI backend** and PostgreSQL data.

## Setup

```bash
cd mobile
cp .env.example .env
# Set EXPO_PUBLIC_API_URL to your PC Wi-Fi IPv4 (ipconfig). Start backend with:
#   python -m uvicorn main:app --host 0.0.0.0 --port 5000 --reload
npm install
npm start
```

Backend in this repo runs on **port 5000** (`uvicorn main:app --port 5000`).

## Architecture

| Layer | Path |
|-------|------|
| HTTP client | `services/api.ts` — axios + SecureStore JWT + 401 logout |
| Domain services | `services/auth.ts`, `hospitals.ts`, `doctors.ts`, `appointments.ts`, `labs.ts`, `bloodbanks.ts`, `records.ts`, `specialities.ts` |
| Mappers (no mock data) | `services/mappers.ts` |
| React Query hooks | `hooks/useMe.ts`, `useHospitals.ts`, `useDoctors.ts`, … |
| Staff dashboard API | `services/staffApi.ts` |

## API mapping (spec → FastAPI)

Your spec paths are aliased to the **actual** backend routes:

| Spec | FastAPI |
|------|---------|
| `POST /auth/login` | `POST /api/user/login` |
| `POST /auth/register` | `POST /api/user/register` |
| `GET /auth/me` | `GET /api/user/get-profile` |
| `POST /auth/upload-profile-pic` | `POST /api/user/update-profile` (multipart `image`) |
| `GET /hospitals` | `GET /api/hospital-tieup/list` |
| `GET /hospitals/:id` | `GET /api/hospital-tieup/details/:id` |
| `GET /doctors` | `GET /api/doctor/list` + `/api/hospital-tieup/public/doctors` |
| `GET /doctors/:id` | `GET /api/doctor/:id` |
| `GET /appointments/my` | `GET /api/user/appointments` |
| `POST /appointments` | `POST /api/user/book-appointment` |
| `GET /appointments/slots` | `GET /api/ai/doctor-slots/:docId` |
| `GET /labs` | `GET /api/lab/list` |
| `GET /blood-banks` | `GET /api/blood-bank/list` |
| `GET /records/counts` | Derived from `GET /api/user/health-records` |
| `GET /specialities` | `GET /api/specialty/public/all` |

Fields use API shapes (`image`, `speciality`, `slotDate`, `open_now`, `available_blood`, etc.). Ratings/reviews only show when present in the database.

## Sharing with a Friend

If you want to zip or share this `mobile` folder with a friend, you should **delete** the following auto-generated/heavy folders before zipping to save space:

- `node_modules/` (Very large, generated by npm)
- `.expo/` (Auto-generated by Expo)
- `.env` (Keep your private keys safe! Your friend can use `.env.example` to make their own)

**What your friend needs to install & run:**

1. Extract the project folder and open a terminal inside the `mobile` directory.
2. Run `npm install` to install all the required modules.
3. Copy `.env.example` to a new file named `.env` and configure the `EXPO_PUBLIC_API_URL` to point to the backend.
4. Run `npm start` (or `npx expo start`) to launch the app!
